Docker Files Explained

Society Maintenance Agent — Dockerfile & docker-compose.yml walkthrough

Overall Architecture

Your machine ┌──────────────────────────────────────────────────────┐ │ │ │ society-agent/data/ ←── your Excel files live here │ │ Maintenance.xlsx │ │ Expenses.xlsx │ │ │ │ │ │ mounted read-only into both containers │ │ ▼ │ │ ┌─────────────────┐ ┌──────────────────────┐ │ │ │ society-api │ │ society-ui │ │ │ │ FastAPI :8000 │◄────│ Streamlit :8501 │ │ │ │ MCP server │ │ (waits for api │ │ │ └─────────────────┘ │ to be healthy) │ │ │ └──────────────────────┘ │ │ │ │ │ agent-memory volume │ │ (SQLite memory.db) │ └──────────────────────────────────────────────────────┘

Both containers run from the same Docker image (society-agent:latest) — just with different startup commands.

Dockerfile — Line by Line

Stage 1: Builder build-only

# Uses a temporary container just to install Python packages
FROM python:3.12-slim AS builder
WORKDIR /app
COPY requirements.txt .
RUN pip install --prefix=/install -r requirements.txt
# All packages land in /install — not in the final image yet

This is a multi-stage build. pip runs here in isolation so the final image doesn't contain pip, build tools, or caches — keeping it small.

Stage 2: Runtime final image

# Fresh slim Python — no pip, no build junk
FROM python:3.12-slim
WORKDIR /app

# Copy only the compiled packages from stage 1
COPY --from=builder /install /usr/local

# Copy all source code (respects .dockerignore)
COPY . .

# Create non-root user for security
RUN addgroup --system appgroup && adduser --system appuser ...
USER appuser

# Expose both ports (only one is actually used per container)
EXPOSE 8000 8501

# Default CMD — overridden by docker-compose per service
CMD ["streamlit", "run", "ui/app.py", ...]
What .dockerignore excludes: The .dockerignore file prevents .venv/, __pycache__/, *.db, .env, and data/ from being baked into the image. Data files are mounted at runtime instead.

volumes — agent-memory

volumes:
  agent-memory:
    driver: local

Docker manages this volume. It stores agent/memory.db — the SQLite file that holds all conversation history. If you remove the container, the memory survives. If you remove the volume, memory is wiped.

How to Change the Data Files

Key insight: The Excel files are never copied into the image. They are mounted from your local society-agent/data/ folder at runtime via the volume ./data:/app/data:ro. This means you can update data without rebuilding the image.

Option 1 — Replace files while containers are stopped (safest)

1
Stop everything: docker compose down
2
Replace society-agent/data/Maintenance.xlsx and/or Expenses.xlsx with your new files (keep the same filenames)
3
Start again: docker compose up

Option 2 — Hot-swap while running (no restart needed)

1
Replace your local Excel files — the volume mount is live, so the container sees the new files immediately on the next request. No restart needed.
2
The :ro (read-only) flag only prevents the container from writing to data/ — you can still write from your host machine normally.
Rename caution: If you rename the files (e.g. Maintenance_v2.xlsx), data_loader.py won't find them. The loader hardcodes the filenames Maintenance.xlsx and Expenses.xlsx. Either keep the names, or update server/data_loader.py and rebuild the image.

Option 3 — Mount a different folder entirely

If your data lives somewhere else on your machine, change the volume line in docker-compose.yml:

# Before
volumes:
  - ./data:/app/data:ro

# After — point to any folder on your machine
volumes:
  - C:/MyData/SocietyFiles:/app/data:ro

No rebuild needed — just docker compose up again.
